# Native CI problems reports

This directory stores the latest diagnostics generated by the
`Build Native Android APKs` workflow when the build fails.

For manual runs (`workflow_dispatch`), keep `persist_problem_report=true` to
auto-commit diagnostics. For `push` runs on repository branches, diagnostics
are persisted automatically on failure (pull requests are excluded).

Current latest report paths:
- `latest/problems-report.html` (if Gradle generated the Problems Report)
- `latest/gradle-build.log` (raw build log from assemble step)
