# Native CI problems reports

This directory stores the latest diagnostics generated by the
`Build Native Android APKs` workflow when the build fails.

For manual runs (`workflow_dispatch`), diagnostics are auto-committed by
default (`persist_problem_report=true`).
For `push` and `pull_request` runs, the workflow also attempts to persist
diagnostics on failure; if repository permissions are read-only, artifacts are
still uploaded and the push step is skipped.

Current latest report paths:
- `latest/problems-report.html` (if Gradle generated the Problems Report)
- `latest/gradle-build.log` (raw build log from assemble step)
- `latest/error-summary.md` (filtered key error lines for quick triage)

Additionally, on failure the workflow writes a condensed error block to the
GitHub job summary (`$GITHUB_STEP_SUMMARY`) so root cause lines are visible
without downloading artifacts.
